# 🆘 Soporte

Gracias por usar Human Genome Labs. Aquí encontrarás el canal adecuado según lo que necesites.

## Antes de preguntar

1. Revisa la [guía de arranque](GETTING_STARTED.md) y la [FAQ](docs/FAQ.md).
2. Busca términos en el [glosario](docs/GLOSSARY.md).
3. Recorre el [índice de documentación](docs/INDEX.md) por tu perfil.

## Dónde pedir ayuda

| Necesito… | Canal |
|---|---|
| 🐛 Reportar un error | Abre un **issue** con la plantilla de *bug report* |
| 💡 Proponer una capacidad | Abre un **issue** `proposal:` (ver [roadmap](docs/ROADMAP.md#-cómo-influir-en-el-roadmap)) |
| 🧬 Cambio de contrato o modelo de datos | Sigue el [Proceso RFC](docs/RFC_PROCESS.md) |
| ❓ Duda de uso | Abre un issue de pregunta o usa **Discussions** si están habilitadas |
| 🔒 Vulnerabilidad de seguridad | **No** abras un issue público: sigue [SECURITY.md](SECURITY.md) |

## Cómo escribir un buen reporte

Incluye siempre:

- Sistema operativo, versión de **Node** (`node -v`) y de **pnpm** (`pnpm -v`).
- El **comando exacto** y el archivo de entrada (o un ejemplo mínimo de `examples/`).
- La salida esperada frente a la obtenida (pega el error completo).
- Si aplica, el resultado de `pnpm check`.

## Qué NO es soporte

Este proyecto **no** ofrece asesoría clínica, diagnóstica ni interpretación médica de datos genómicos. Para decisiones de salud, consulta a un profesional.
